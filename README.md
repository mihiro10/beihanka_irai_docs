# 🍚 米飯課への製造依頼 (AppSheet System Documentation)

## 📌 プロジェクトの目的 (What is this system?)
「米飯課への製造依頼」アプリケーションは、食品製造工場における**「生産管理部門」と「米飯課（製造現場）」間のスケジュール調整とタスク管理をデジタル化するAppSheet専用アプリ**です。

従来のアナログなやり取りによるスケジュールの重複や直前での変更トラブルを防ぐため、以下のビジネスロジックをシステムで強制・一元管理しています：
*   **キャパシティ管理:** 1日あたり「早朝」「昼間」の2つの時間帯で、それぞれ規定された上限分数（例: 180分）が存在します。
*   **米飯課優先の予定ブロック:** 米飯課が事前に「清掃」や「ミーティング」などの予定時間をブロック（登録）すると、その分だけ生産管理側が依頼可能な**空き時間枠**が統合稼働グラフ上で動的に減少します。
*   **4日ルールのロック機能:** 製造日の4日以内に入った依頼は米飯課によって**【確定】**状態となり、自動的に編集・削除がロックされます。直前にどうしても変更が必要な場合は「管理者日程調整」による強制リスケと、それに伴う全自動の監査ログ（ChangeLog）追記が必須となるフェイルセーフ設計です。

## 🎯 このGitHubリポジトリの役割 (Why does this repo exist?)
**AppSheet自体には「ソースコード」や「Gitを用いたバージョン管理」の概念が存在しません。**
そのため、数年後にこのアプリの改修を行おうとしたエンジニア（またはAIアシスタント）が、「どの数式がどこに影響しているのか」「このSlice（仮想ビュー）やActionは何のビジネスロジックのために動いているのか」といった全体設計の全貌を見失うという、致命的なブラックボックス化問題が発生します。

本リポジトリは、**「AppSheetアプリの設計書・仕様・ビジネスロジックをGitで完全にバージョン管理し、常に最新のシステムアーキテクチャを未来の担当者（人間・AI問わず）へ安全に引き継ぐこと」**を唯一の目的として作られた特殊形態のリポジトリです。

## 📂 ディレクトリ構成 (Repository Structure)

*   **[`/user_guides/`](user_guides/)**
    *   現場のタブレット操作者（生産管理・米飯課）に向けた純粋な運用マニュアルです。技術的な文脈を排除した操作手順書が保管されています。
*   **[`/appsheet_architecture/`](appsheet_architecture/)**
    *   このアプリの「ソースコード」に相当する**超高解像度の設計書群**です。データベースのテーブル定義、仮想カラム式（App Formula）、UIレイアウト（Views）、および自動化ルール（Actions/Slices）が、すべての依存関係や開発担当者によるビジネスロジック解説（Context）と共に記録されています。改修を行うエンジニアはまずここを読んでください。
*   **[`/automation_tools/`](automation_tools/)**
    *   AppSheet側のGUI変更結果を、GitHubの仕様書（Markdown）へ自動同期するための専用のPythonパーサー・ツール群です。
*   **[`/archive/`](archive/)**
    *   過去の要件定義書（PRD）や、検証用にAppSheetのAPIから直接抽出した過去データ（JSON形式の各テーブルローカルバックアップ）などのアーカイブ置き場です。

## 🛠 開発者向け: リポジトリの運用方法 (How to work with this repo)
新しい開発者（またはAI）がこのAppSheetアプリを安全かつ適切に改修・メンテナンスする際のワークフローは以下の通りです。

1.  **① 既存仕様・影響範囲の把握**  
    AppSheetのエディタを開く前に、まずは `appsheet_architecture/` 内の設計書やER図を読み込みます。目的の変更箇所（例: 4日ルールの変数の変更、稼働グラフカラムの追加）が他にどう影響しているかを完全に把握してください。
2.  **② AppSheetエディタでのGUI改修**  
    実際にAppSheetのブラウザエディタ上でカラム設定、App Formula、Action、Sliceを追加・修正します。
3.  **③ 仕様書（当リポジトリ）への自動同期**  
    AppSheet上で仕様変更を行った後は、必ず **[`automation_tools/README.md`](automation_tools/README.md)** の手順に従い、専用のPythonスクリプトを実行してください。
    
> 💡 **自動同期ツールの使い方:**  
> Pythonスクリプトを実行するだけで、AppSheetの最新のHTML定義をスクレイピングし、`appsheet_architecture/` 内のMarkdown設計書群へ自動出力します。このツールの非常に強力なところは、**過去の開発者がMarkdown上に手書きで残した「人間向けの仕様コメント（Context）」を一切破壊することなく、完全に新しいテーブル構成や数式部分だけをマージ・再構築してくれる点**です。
> 自動生成された差分をGitで `commit` ＆ `push` すれば、改修作業は無事完了です。
