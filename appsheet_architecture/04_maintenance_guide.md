# 🔧 保守・運用ガイド (For Developers / Admins)

本ドキュメントは、システム管理者や将来の開発者・AIがAppSheetのメンテナンスを行う際の手順や注意点をまとめています。

## 1. ユーザー（Users）の追加・ID管理設計
現場は共有タブレット（同じEmailアドレス）を使用するため、システム標準のメールアドレスによる識別（USEREMAILの直接利用）は意図的に排除されています。
1. 新規ユーザーを追加する場合、必ず `Users` テーブルに登録します。
2. その際、一意の **`UserId` (UNIQUEID())** がキーとして生成・付与されます。
3. `Email` 列のデータは重複してもシステム上エラーにはなりません。

## 2. 依頼元（Requester）のドロップダウン設定
各フォーム（`米飯課予定登録` 等）の「依頼元」選択欄は `Users` テーブルを参照していますが、`Ref` タイプは使用していません。
*   カラムタイプは **`Enum` (Base Type: Text)** に設定されています。
*   `Valid If` に `Users[FullName]` を指定することで、名前をリスト化して提示し、選択された「文字列」のみを直接保存する仕様を取っています（キー参照エラーを回避するためです）。

## 3. カレンダーの「14日前ブロック」バリデーション
米飯課の予定登録フォームには、過去または14日以内の直近の日付を無効化する、厳密な `Valid If` ロジックが実装されています。
*   **実装場所:** `Requests` テーブルの ` TargetDate (製造日) ` カラム
*   **実装式:**
```excel
IF(
  LOOKUP([_THISROW].[Requester], "Users", "FullName", "Department") = "米飯課",
  [_THIS] >= (TODAY() + 14),
  TRUE
)
```
*   **注意点:** 判定は `Role` ではなく `Department` カラムを参照しています。

## 4. エラーメッセージ・システム文言のカスタマイズ
### ロジックエラー文言
`TargetDate` で14日以内の制限に引っかかった場合は、利用者を迷わせないよう、`Invalid value error` 式にて専用の案内文章（"14日以内の追加は、生産管理へ直接ご連絡ください。"）が表示されるように制御されています。

### 基本システム文言 (Localization)
キャンセル確認時などに出現する標準の `No` ボタンなどは、AppSheetの **Settings > UX > Localize** 設定から `いいえ` 等に手動設定されています。変更の際は、数式モード（ビーカーアイコン）ではなくテキストモード（Tアイコン）になっている場合、**ダブルクォーテーションを付けない** よう注意してください。
